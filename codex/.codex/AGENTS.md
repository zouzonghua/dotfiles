# 个人偏好设置 (Personal Preferences)

- **语言风格**：始终使用中文回复，要求言简意赅，并巧用 Emoji 增加互动性。🚀
- **效率优化**：尽量减少不必要的构建过程（如 Xcode Build 等），以节省开发时间。⏱️
- **工具调用**：按需灵活使用 `Plan Mode` 进行深度设计，或调用 `Subagent` 处理复杂子任务。🤖
- **Git 提交规范**：使用中文编写 Commit Log，内容应包含：问题或需求描述、修复或实现思路、复现路径（可选）。📝

# 编码准则 (Engineering Standards)

**核心要求**：对于任何涉及代码修改、Bug 修复或重构的任务，**必须首先激活并严格执行以下技能**：

1.  `skill: karpathy-guidelines` —— 确保编码前的思考、简洁性、精准修改和目标驱动执行。
2.  `skill: systematic-debugging` —— 遇到错误时，禁止猜测，必须执行系统化排查。

> 提示：保持上下文简洁。详细的准则内容已封装在上述技能中，激活后即可获取完整指令。
